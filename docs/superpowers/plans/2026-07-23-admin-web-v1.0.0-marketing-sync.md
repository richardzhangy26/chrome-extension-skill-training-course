# Admin Web v1.0.0 官网内容同步 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 把 `admin_web` 官网首页/导航/页脚文案同步到扩展 v1.0.0,关闭博客入口,新增"问题反馈"(Gitee Issues)与 Chrome 应用商店提示,下载链接改指向 Gitee v1.0.0 ZIP。

**Architecture:** 纯配置与文案改动,不引入新组件、新路由或新依赖。改动集中在 `admin_web/src/config/*`、`admin_web/src/components/{layout,blocks}/*` 和 `admin_web/project.inlang/messages/{zh,en}.json` 三类文件。

**Tech Stack:** TanStack Start + React 19 + `@tanstack/react-router` 1.159.5 + `@inlang/paraglide-js`(编译期生成 `src/locale/paraglide/**`,不手动改)+ Biome。

**对应设计文档:** `docs/superpowers/specs/2026-07-23-admin-web-v1.0.0-marketing-sync-design.md`

## Global Constraints

- 所有文案键改动必须同时更新 `admin_web/project.inlang/messages/zh.json` 和 `en.json` 两侧,不能只改一侧。
- 不手动编辑 `admin_web/src/locale/paraglide/**`(Vite 插件在 `pnpm dev`/`pnpm build` 时自动从 `project.inlang/messages/*.json` 编译生成)。
- 不新增/编造 `home_testimonials_*` 内容。
- 不更换 `features.tsx` 五张截图和 hero 产品大图(Cloudflare R2 资源,本次无上传权限)。
- 不处理 `/changelog` 页面(`content/changelog/*.md`),该页面是孤立的模板占位内容,导航/页脚未链接它。
- `DOWNLOAD_URL` 指向 Gitee **v1.0.0 具体 tag** 页(`.../releases/tag/v1.0.0`),不是"latest"别名;后续发新版本需手动更新这个常量,与根目录 `README.md` 维护方式一致。
- `admin_web` 没有配置单元测试框架;每个任务的"测试"就是 `pnpm build`(产物构建,能捕获 TS 类型错误和 paraglide 消息键缺失,这是主要判定标准)。所有命令都在 `admin_web/` 目录下执行。
- **`pnpm check`(Biome)基线现状**:在本计划开始前,worktree 里对已提交代码跑 `pnpm check` 就有 **48 个预存在的格式化错误**,与本计划无关,其中包括本计划要改的 `hero.tsx`、`stats.tsx`、`links.ts`、`navbar-config.ts`、`website.ts` 等文件本身。因此每个任务不要求 `pnpm check` 整体零错误,只要求:自己编辑/新增的那几行不引入新的诊断;**不要**顺手用 `--write` 或手动修复这些文件里其他与本任务无关的历史格式问题(会引入大量无关 diff,超出任务范围)。
- 不执行 `pnpm deploy`,除非用户明确要求上线。
- `project.inlang/messages/{zh,en}.json` 里的 key 按字母序排列,新增 key 要插入到正确的字母序位置(每个任务里已给出具体插入点)。
