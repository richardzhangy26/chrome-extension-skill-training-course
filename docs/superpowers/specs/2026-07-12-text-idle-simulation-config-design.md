# 文本模式空闲态显示对话模拟配置

## 目标

文本模式尚未开始对话时，在开始按钮上方显示完整的“对话模拟 / 知识库”配置栏，使用户无需进入对话即可打开配置弹窗或切换两个功能。

## 设计

- 在 `SidePanel` 的文本模式 `IDLE` 分支中复用现有 `SimulationConfigBar`。
- 配置栏沿用当前 `simulationConfig` 状态以及现有的打开弹窗、切换模拟对话、切换知识库回调。
- 配置栏位于 `StartButton` 上方，布局与语音模式开始前的底部操作区一致。
- `StartButton` 的行为和 `trainTaskId` 限制保持不变。
- 对话开始后继续由 `ChatInput` 显示现有配置栏，不新增存储或运行时分支。

## 验证

- 自动化检查文本模式空闲态会渲染配置栏和开始按钮。
- 自动化检查非空闲状态不会新增重复配置栏。
- 运行侧边栏 lint 和 TypeScript 类型检查。

## 非目标

- 不修改语音模式。
- 不修改对话模拟、知识库的存储结构或 LLM 注入逻辑。
- 不调整现有聊天输入区样式和交互。
