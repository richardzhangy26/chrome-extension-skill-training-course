# 训练模式下拉框设计

## 目标

将侧边栏 Header 中现有的“文字/口语”二态切换按钮改为三项下拉框，为后续“能力训练 Pro”功能预留稳定入口，同时不改变当前文字训练和口语训练的业务行为。

下拉框包含以下选项：

- 能力训练：对应现有文字训练。
- 口语训练：对应现有口语训练。
- 能力训练 Pro：本期仅提供可选择入口，暂时复用现有文字训练界面。

## 组件与状态

沿用 `ModeToggle.tsx` 的组件边界，将它从二态按钮调整为受控下拉组件，继续由 `SidePanel.tsx` 管理当前训练模式。

`TrainingMode` 从 `text | voice` 扩展为 `text | voice | pro`。组件接收当前模式、切换回调和禁用状态，负责展示当前选项以及下拉菜单，不承载训练业务逻辑。

Header 标题根据模式显示：

- `text`：能力训练助手。
- `voice`：口语训练助手。
- `pro`：能力训练助手（Pro 标识由下拉框展示，避免窄侧边栏标题换行）。

## 模式行为

`voice` 继续进入现有口语训练分支；`text` 和 `pro` 均进入现有文字训练分支。这样 Pro 的选择状态是真实且可观察的，后续实现独立功能时可以直接增加 `pro` 内容分支，无需再次修改选择器接口。

切换模式时沿用当前清理规则：重置普通文字、多角色和语音训练状态，避免旧会话泄漏到新模式。训练进行中继续禁用模式选择，防止中途切换破坏状态机。

现有 `llmConfigStorage.voiceModeEnabled` 继续仅表示“默认以口语模式启动”。选择 `voice` 时写入 `true`，选择 `text` 或 `pro` 时写入 `false`。本期不增加 Pro 持久化字段，因此重新打开侧边栏后按现有默认规则进入能力训练或口语训练。

## 交互与样式

选择器沿用 Header 现有的半透明白色、圆角、模糊背景和 hover/disabled 风格。下拉菜单使用清晰的白色面板与高对比文字，当前项提供选中标识。

组件支持：

- 点击触发器打开或关闭菜单。
- 点击选项完成切换并关闭菜单。
- `Escape` 关闭菜单。
- 点击组件外部关闭菜单。
- 训练进行中禁用触发器，并保留现有禁用原因提示。
- 合理的 ARIA 属性和键盘焦点，满足现有 ESLint 可访问性约束。

## 异常与边界

- 选择当前模式时不触发重置或存储写入。
- 禁用状态下不能打开菜单或切换选项。
- Pro 暂时复用文字训练，不展示空白页或“敬请期待”阻断页。
- 不修改文字、语音、多角色训练 hook，也不新增后端、权限或存储 schema。

## 验证

- 检查三个选项均能显示并正确更新 Header 标题。
- 检查 `text` 与 `pro` 使用文字训练界面，`voice` 使用口语训练界面。
- 检查切换会清理旧会话，忙碌状态会禁用选择器。
- 检查下拉菜单的点击、外部关闭和 `Escape` 行为。
- 运行 `pnpm -F @extension/sidepanel lint`。
- 运行 `pnpm -F @extension/sidepanel type-check`。
- 运行根工作区生产构建，确认扩展能够正常打包。

## 非目标

- 不实现能力训练 Pro 的独立业务逻辑。
- 不持久化 Pro 为默认启动模式。
- 不修改现有训练状态机、配置同步或 Admin Web。
